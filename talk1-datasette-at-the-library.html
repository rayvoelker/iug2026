What is Datasette?

  • Created by Simon Willison (co-creator of Django)
  • "A tool for exploring and publishing data"
  • Aimed at journalists, museum curators, archivists, local governments, and anyone else who has data
  • Read-only by default — safe to hand to anyone

Pre-Built Queries for Domain Experts

  • Not everyone wants (or needs) to write SQL
  • Datasette supports "canned queries" with parameter inputs
  • Examples:
    • "Items at [branch] not checked out since [date]"
    • "Top circulating titles by [format]"
    • "Collection age distribution for [branch]"
  • Domain experts get answers; developers maintain the queries

SQL Example: Location Analysis


select
  item.location_code,
  l.location_name,
  l.branch_name,
  count(*) as count_items
from
  item
  join location_view as l
    on l.location_code = item.location_code
where branch_name = :branch_name
group by 1, 2, 3
Named parameters like :branch_name give users a nice input form in Datasette without writing SQL.

Try It Yourself — 5 Minutes


# Install
pip install datasette sqlite-utils

# Got a CSV? Make it a database.
sqlite-utils insert my-data.db my_table my-data.csv --csv

# Enable full-text search
sqlite-utils enable-fts my-data.db my_table title description

# Launch
datasette my-data.db
# → http://localhost:8001/

Or try Datasette Lite — no install needed!
