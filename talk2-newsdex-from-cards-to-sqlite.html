65 Years of Index Cards

1927 — 1992

  • Librarians hand-typed index cards for newspaper articles
  • Each card: title, newspaper, date, page, column, subject, image indicator
  • Arranged by subject in the card catalog
  • Coverage: Enquirer, Post, Herald, Cincinnati Magazine, Business Courier, Downtowner
  • Selected coverage reaching back to the early 1800s
Emphasize the human element: every card represents a librarian reading an article and deciding it was worth preserving a reference to. The subject assignments are particularly valuable — expert human classification, not automated tagging.

Newsdex Goes Digital (1992)

  • Card catalog computerized in the early 1990s
  • March 1992: newspaper index becomes "Newsdex"
  • Provides citations — not full text
  • Staff continue adding entries, including retroactive indexing pre-1900
  • No library card required — genuinely open data

Reading MARC with pymarc


from pymarc import MARCReader

with open('newsdex-full-marc-export.mrc', 'rb') as fh:
    reader = MARCReader(fh)
    for record in reader:
        title = record['245']['a'] if record['245'] else None
        newspaper = record['260']['b'] if record['260'] else None
        date = record['260']['c'] if record['260'] else None

        subjects = []
        for field in record.get_fields('650'):
            subjects.append(field['a'])
Walk through the defensive coding pattern: not every record has every field. The repeatable fields (650 subjects) are particularly interesting — they represent the librarians' classification work across decades.

The Translation Layer

MARC Subfield Becomes Notes
245 $a title Article title
260 $b newspaper Publication name
260 $c date Needs normalization!
300 $a page, column, has_image Parse "7:2 pic"
650 $a subjects Repeatable (JSON array)
008 pos 7-10 year For sorting/faceting

This mapping requires understanding of both the MARC standard and decades of Newsdex cataloging conventions.

Into SQLite with sqlite-utils


import sqlite_utils

db = sqlite_utils.Database("newsdex.db")

# Insert records
db["articles"].insert_all(records, pk="record_id")

# Enable full-text search
db["articles"].enable_fts(["title", "subjects", "notes"])

# Create indexes for faceting
db["articles"].create_index(["newspaper"])
db["articles"].create_index(["year"])

Lessons Learned

  1. Domain expertise is irreplaceable
    Developers + librarians together > either alone
  2. "Good enough" ships; "perfect" doesn't
    Get the data out there and iterate
  3. The presentation layer matters
    Same data in MARC vs. Datasette tells completely different stories
  4. Historical data has compounding value
    Every year, the older entries become more valuable
